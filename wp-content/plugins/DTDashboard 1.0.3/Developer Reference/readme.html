DTDashboard - WordPress Plugin Overview

Plugin Information

Plugin Name: DTDashboard
Plugin URI: https://www.statureit.com/
Description: This Plugin has some widgets that show statistics about the website.
Version: 1.0.3
Author: Stature IT
Author URI: https://www.statureit.com/

Overview

The "DTDashboard" is a WordPress plugin developed by Stature IT. It provides various widgets for displaying statistics and information about a website. The plugin is designed to enhance the WordPress dashboard by adding widgets that offer insights into website performance and engagement.

Key Features

  • Adds a custom dashboard menu item called "DTDashboard" to the WordPress admin menu.
  • Enqueues necessary scripts and styles for rendering widgets and charts.
  • Registers multiple custom widgets that display different types of statistics and data.
  • Organizes and renders these widgets within the WordPress dashboard.

Widget Categories

  • Courses
  • Member Growth
  • Membership Status
  • Member Rank
  • Member Location
  • Member Languages
  • Groups
  • Posts With Most Engagement
  • Flagged Words

Purpose

The "DTDashboard" plugin aims to provide website administrators and content managers with valuable insights into their website's performance and engagement. It offers a convenient way to access important statistics and data directly from the WordPress dashboard.

Conclusion

This plugin enhances the WordPress experience by simplifying the process of accessing and visualizing essential website data within the WordPress dashboard.

DTDashboard.php

Overview

The DTDashboard.php file is the main file of the DTDashboard plugin. It serves as the entry point for the plugin and defines various actions and functions to create and render the DTDashboard in the WordPress admin area.

Plugin Information

  • Plugin Name: DTDashboard
  • Plugin URI: https://www.statureit.com/
  • Description: This Plugin has some widgets that show statistics about the website.
  • Version: 1.0.3
  • Author: Stature IT
  • Author URI: https://www.statureit.com/

Key Functions and Actions

  1. add_action('admin_menu', 'create_dt_dashboard')
    • Description: Adds a menu page for the DTDashboard in the WordPress admin menu.
    • Callback: create_dt_dashboard
  2. add_action('admin_enqueue_scripts', 'enqueue_dt_dashboard_styles_and_scripts')
    • Description: Enqueues necessary scripts and styles for the DTDashboard.
    • Callback: enqueue_dt_dashboard_styles_and_scripts
  3. require_once plugin_dir_path(__FILE__) . 'Widgets.php'
    • Description: Includes the Widgets.php file, which registers and defines custom widgets for the dashboard.
  4. add_action('widgets_init', 'register_dt_widgets')
    • Description: Registers the custom widgets defined in the Widgets.php file.
    • Callback: register_dt_widgets
  5. function render_dt_dashboard()
    • Description: Renders the DTDashboard content in the WordPress admin area.
    • Content: The DTDashboard consists of various widgets that display statistics about the website. These widgets are organized into different columns on the dashboard.

Dashboard Widgets

The DTDashboard plugin adds the following widgets to the WordPress admin dashboard:

  • Courses: Displays information about courses.
  • Member Growth: Shows statistics related to member growth.
  • Membership Status: Displays membership status information.
  • Member Rank: Provides information about member rankings.
  • Member Location: Shows member locations on a map.
  • Member Languages: Displays information about member languages.
  • Groups: Provides information about groups.
  • Posts With Most Engagement: Shows posts with the most engagement.
  • Flagged Words: Displays flagged words.

  • dt-dashboard-styles (Version 1.0.1): Custom CSS styles for the DTDashboard.

Widgets.php

Overview

This PHP file serves as a central repository for defining multiple widget classes used within a WordPress plugin. It consolidates these widget classes into a single file for organizational and maintainability purposes. Each class represents a distinct widget designed to offer specific functionalities or display particular types of information on the WordPress site.

Purpose

The primary purpose of this file is to centralize the management of widget classes, making it easier for developers to maintain and extend the plugin's functionality. By grouping related widgets in a single file, it enhances code organization and readability.

Advantages

  • Modularity: Keeping widget classes in one file promotes modularity, allowing developers to add, modify, or remove widgets with ease.
  • Organization: It helps in organizing the codebase, especially when dealing with numerous widgets in a complex plugin.
  • Maintainability: Centralizing widget classes simplifies maintenance tasks, such as bug fixes and updates.
  • Code Reusability: Developers can reuse these widget classes across different sections of the plugin or even in other projects.

Usage

To utilize these widget classes, they are typically instantiated within the WordPress plugin. Each widget class may define its unique set of functionalities and data presentation tailored to the specific needs of the plugin.

Further Details

For detailed information about individual widget classes and their functionalities, refer to the respective class definitions within this document.

BaseWidget.php

Overview

The Base_DT_Widget abstract class serves as the foundation for creating custom widgets in the DTDashboard plugin. It extends the WP_Widget class and provides common functionality and AJAX callbacks for widgets.

Constructor

The constructor of Base_DT_Widget initializes the widget with essential parameters.

  • Parameters:
    • $widget_id: Unique ID for the widget.
    • $widget_name: Name of the widget.
    • $widget_description: Description of the widget.

Actions

The Base_DT_Widget class defines several AJAX actions that can be used by widgets for handling asynchronous requests.

  • table_content(): Handles AJAX requests for table content.
  • widget_content(): Handles AJAX requests for widget content.

Widget Method

The widget() method is responsible for rendering the widget's content in the WordPress admin area.

  • Parameters:
    • $args: Widget arguments and container details.
    • $instance: Widget instance settings.

Abstract Method

The Base_DT_Widget class defines an abstract method initial_widget_content() that should be implemented by child widgets. This method is used to render the initial content of the widget.

Utility Methods

The class provides utility methods to sanitize and retrieve widget instances.

  • get_widget_instance_by_id($widget_id): Retrieves a widget instance by its ID.

Courses.php

Overview

The Courses class is a custom widget class that extends the Base_DT_Widget class. It is designed to display statistics about courses in the DTDashboard plugin. This widget calculates and displays data about course completion times, creation dates, and user counts.

Constructor

The constructor of the Courses class initializes the widget with the necessary parameters.

  • Parameters:
    • $widget_id: Unique ID for the widget.
    • $widget_name: Name of the widget.
    • $widget_description: Description of the widget.

Widget Methods

The Courses class defines several methods for rendering widget content and retrieving data:

  • initial_widget_content(): This abstract method is used to render the initial content of the widget. Child classes should implement this method.
  • getWidgetContent(): Generates and returns HTML content for the widget, including course statistics for different time periods.
  • generateCourseList($data): Generates HTML for displaying a list of courses with completion times, including the top 3 fastest and slowest completion times.
  • getTableData(): Generates an HTML table with course statistics, including course name, average completion time, creation date, completed user count, and in-progress user count.
  • getData($start_date, $end_date): Retrieves course data from the database, including completion times, creation dates, completed user counts, and in-progress user counts.

Developer Notes

The Courses class provides a structured way to create a widget that displays course statistics in the DTDashboard plugin. Developers can extend this class and implement the initial_widget_content() method to customize the widget's appearance and behavior.

DTScript.js

Overview

The provided JavaScript code defines custom functionality for WordPress widgets within the admin dashboard. It allows users to interact with widgets in various ways, such as clicking on widget headers to create popups and toggling panel visibility.

Functions and Methods

createPopup(widgetTitle, sortby = null, order = null)

Creates a popup for a widget when a widget header is clicked. The popup contains specific content related to the widget and its controls.

  • widgetTitle: The title of the widget for which the popup is created.
  • sortby: Optional sorting parameter for table data.
  • order: Optional order parameter for table data sorting.

createFields(widgetTitle, container, tableContainer, sortby = null, order = null)

Dynamically creates input fields and controls within a widget popup based on the widget's title. This function is used to customize the popup content for different widget types.

  • widgetTitle: The title of the widget for which fields are created.
  • container: The container element to which fields and controls are appended.
  • tableContainer: The container element for the data table.
  • sortby: Optional sorting parameter for table data.
  • order: Optional order parameter for table data sorting.

createTable(widgetTitle, container, selectedYear = null, sortby = null, order = null)

Creates an AJAX request to retrieve data and generates a data table within a widget popup. The content and structure of the table depend on the widget type and parameters.

  • widgetTitle: The title of the widget for which the table is created.
  • container: The container element in which the table is appended.
  • selectedYear: Optional selected year parameter for data filtering.
  • sortby: Optional sorting parameter for table data.
  • order: Optional order parameter for table data sorting.

updateTable(widgetTitle, container, selectedYear = undefined, fromDate = undefined, toDate = undefined)

Updates the data table within a widget popup based on user interactions, such as selecting a year or date range. Performs an AJAX request to fetch updated data and refreshes the table.

  • widgetTitle: The title of the widget for which the table is updated.
  • container: The container element containing the table to be updated.
  • selectedYear: Optional selected year parameter for data filtering.
  • fromDate: Optional "From" date parameter for data filtering.
  • toDate: Optional "To" date parameter for data filtering.

updateWidgetData(postbox)

Updates widget data within a postbox element, including the ability to select a year and retrieve new data based on user interactions.

  • postbox: The postbox element containing the widget to be updated.

performAjaxRequest(selectedYear)

Performs an AJAX request to fetch widget content data based on the selected year and updates the widget content accordingly.

  • selectedYear: The selected year for data filtering.

generatePostHtml()

A helper function used to generate HTML for displaying top posts with engagement.

togglePanelVisibility()

Toggles the visibility of a widget panel when the "Toggle panel" icon is clicked.
